Computer Security, Encryption & Authentication

Computer security is the protection of the computer system from any event or action that could cause a loss of or damage to hardware, software, data, or information[cite: 29]. This lesson covers the core principles, encryption mechanics, and structural defenses used to maintain secure systems at the AS/A Level standard.

Learning Objectives

  • 11.1.2.4 Explain the CIA triad, encryption, digital signatures, and certificates
  • 11.1.2.5 Describe authentication methods, access rights, and network security measures (Firewalls, Proxies, IDS)

1. The CIA Triad

The Computer Security Triad (CIA) is a foundational model designed to guide policies for information security within an organization[cite: 33].

Principle Definition Implementation / Purpose
Confidentiality Ensures sensitive information is accessed only by authorized persons[cite: 42]. Implemented using usernames, passwords, Access Control Lists (ACLs), and encryption[cite: 43].
Integrity Ensures data has not been tampered with and is correct, authentic, and reliable[cite: 48, 49]. Protects against unauthorized deletion, addition, or modification[cite: 50]. Implemented using hashing and data encryption[cite: 51].
Availability Ensures information, networks, and resources are up and running for those who need them[cite: 54, 56]. Implemented via backups and redundant systems[cite: 62].

2. Symmetric vs Asymmetric Encryption

Encryption transforms readable plaintext into unreadable ciphertext[cite: 196].

Feature Symmetric Encryption Asymmetric Encryption (Public Key Cryptography)
Definition Uses the same secret key for both encryption and decryption[cite: 188]. Uses a pair of keys: a public key for encryption and a private key for decryption[cite: 218].
Advantages Fast encryption/decryption; efficient for large data volumes[cite: 208, 209]. Eliminates the key distribution problem; increases security as private keys are never transmitted[cite: 228, 229].
Disadvantages Securely sharing the key is a major risk; not scalable for large systems[cite: 213, 214]. Slow process compared to symmetric; inappropriate for bulk message decryption[cite: 233].
Application File/disk encryption (BitLocker), database protection, Wi-Fi (WPA2)[cite: 201, 202, 203]. SSL/TLS (HTTPS), Encrypted email, Cryptocurrencies[cite: 226, 239, 240, 241].

3. Digital Signatures & Certificates

Digital Signatures

A digital signature proves a digital message or document is authentic and hasn't been tampered with[cite: 1]. It includes the original content and a signature block[cite: 5].

  • How it works: The signature contains a hashed version of the document encrypted with the sender’s private key[cite: 6].
  • Authentication: Confirms the message was really sent by the claimed sender[cite: 10].
  • Integrity: Ensures content hasn't been altered after signing[cite: 12].
  • Non-repudiation: The signer cannot deny later that they signed the document[cite: 14].

Digital Certificates

A digital certificate proves the authenticity of a device, server, or user through the use of cryptography and public key infrastructure (PKI)[cite: 244]. Only publicly trusted Certificate Authorities (CAs) can issue them[cite: 252].

Contents of a typical Digital Certificate:

  • Owner’s Name/Organization: Who the certificate is for[cite: 257].
  • Public Key: Used in encrypting data or verifying digital signatures[cite: 258].
  • Issuer (CA): The trusted entity that issued the certificate[cite: 259].
  • Serial Number & Validity Period: Unique ID and start/end dates[cite: 260, 261].
  • Digital Signature: A signature from the CA proving the certificate wasn't altered[cite: 263].

4. Authentication, Authorization & Access Control

Authentication Methods

Authentication verifies that the individual is the person they claim to be[cite: 64]. Multi-Factor Authentication (MFA) requires two or more independent ways to verify identity[cite: 137, 138].

Category Description Examples / Details
Knowledge-based
(Something you know)		 Identifies users by asking for secret information[cite: 83]. Usernames, passwords, PINs, passphrases[cite: 85, 87, 88].
Possession-based
(Something you have)		 Uses physical objects (tokens) the user possesses[cite: 106]. Memory tokens: Store info but do not process it (e.g., Magnetic cards). Cheap, used with PINs [cite: 114, 115].
Smart tokens: Process info, more expensive, difficult to forge[cite: 117, 118].
Biometric-based
(Something you are)		 Translates a personal characteristic into a digital code[cite: 134]. Physiological: Face, fingerprint, hand, iris, DNA [cite: 131].
Behavioral: Keystroke, signature, voice[cite: 131].

Authorization, Access Control & Audit Trails

  • Authorization: Ensures authenticated users access only approved resources[cite: 140]. Takes place after identification and authentication[cite: 141].
  • Access Control: Policies outlining what permission is needed, what data can be accessed, and allowed actions[cite: 146].
  • Audit Trail: Automatically generated records tracking every event (create, delete, read, modified)[cite: 151, 153]. Events are aligned to the user and time-stamped, allowing organizations to pinpoint the source of a breach[cite: 149, 154, 155]. Users cannot alter these logs[cite: 152].

5. Network Security Measures

Technical measures are deployed to protect networks from external and internal threats.

Measure Definition Primary Functions
Firewall Software/hardware that filters information coming through the internet connection[cite: 158]. Protects from unauthorized intrusion; blocks access from the outside[cite: 159, 160].
Proxy Server Hardware/server that acts as a middleman between the user and the web server[cite: 162]. Hides user's real IP address; filters internet traffic; stores frequently used files into CACHE to speed up access[cite: 163, 165, 166].
IDS (Intrusion Detection System) Software built to monitor network traffic and discover irregularities or malicious activity[cite: 176]. Scans for harmful activity or policy breaching; issues alerts to administrators or SIEM systems[cite: 180, 181, 182].

Independent Practice (Cambridge AS/A Level Style)

Apply

A bank allows customers to access their accounts online. Explain how a digital certificate guarantees to the customer that the bank's website is authentic.

Analyze

Compare the functionality of a Firewall with that of a Proxy Server. Explain why an organization might choose to implement both simultaneously.

Evaluate

Evaluate the use of biometric authentication versus possession-based authentication (smart tokens) for securing a high-level corporate server room. Consider aspects of security, cost, and user convenience.

Create

Describe the exact step-by-step cryptographic process a sender uses to apply a digital signature to a document, and the process the receiver uses to verify it, ensuring non-repudiation.

Plenary / Self-Check

Q1: What are the three elements of the CIA triad?

Confidentiality, Integrity, and Availability[cite: 39].

Q2: Why is symmetric encryption not scalable for a very large number of communicating users?

Because each pair of users needs a unique, securely shared secret key. Securely distributing these keys becomes unmanageable at scale[cite: 213, 214].

Q3: How does an Audit Trail assist in maintaining data integrity?

It automatically generates a time-stamped record of every event (create, read, modify, delete) tied to specific users, allowing tracking of any unauthorized changes[cite: 148, 153, 154, 155].
