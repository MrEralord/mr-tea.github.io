LMC Simulator

The Little Man Computer (LMC) is a simplified model of a CPU. It uses a tiny instruction set to demonstrate how assembly language programs work — fetch, decode, execute — just like a real processor.

Conceptual Anchor

The Little Man

Imagine a tiny person in a room with 100 numbered mailboxes (memory), a calculator (ALU), an inbox (input), and an outbox (output). He follows instructions one by one: fetch a number from a mailbox, add it, store the result, output it. That's how a CPU works.

Rules & Theory

LMC Architecture

┌─────────────────────────────────────┐ │ LMC Components │ │ │ │ INBOX ──→ [Accumulator] ──→ OUTBOX │ │ ↕ │ │ ┌─────────────────────┐ │ │ │ Mailboxes 00 – 99 │ │ │ │ (100 memory cells) │ │ │ │ Store instructions │ │ │ │ AND data │ │ │ └─────────────────────┘ │ │ │ │ Program Counter: tracks next │ │ instruction to execute │ └─────────────────────────────────────┘

LMC Instruction Set

Mnemonic Code Description
INP 901 Read input into accumulator
OUT 902 Output the value in accumulator
ADD 1xx Add value at mailbox xx to accumulator
SUB 2xx Subtract value at mailbox xx from accumulator
STA 3xx Store accumulator value in mailbox xx
LDA 5xx Load value from mailbox xx into accumulator
BRA 6xx Branch always — jump to mailbox xx
BRZ 7xx Branch if accumulator = 0
BRP 8xx Branch if accumulator ≥ 0 (positive)
HLT 000 Stop the program
DAT — Reserve a mailbox for data

Worked Examples

1 Add Two Numbers

INP // Read first number → ACC STA FIRST // Store ACC in FIRST INP // Read second number → ACC ADD FIRST // ACC = ACC + FIRST OUT // Output result HLT // Stop FIRST DAT // Reserve mailbox for first number

2 Count Down from Input to 0

INP // Read starting number LOOP OUT // Output current value SUB ONE // ACC = ACC - 1 BRP LOOP // If ACC >= 0, go back to LOOP HLT // Stop ONE DAT 1 // Constant: 1

Common Pitfalls

Forgetting DAT

Every variable/label needs a DAT declaration at the end of the program. Without it, the LMC doesn't know where to store your data.

BRZ vs BRP Confusion

BRZ branches only when ACC = 0. BRP branches when ACC ≥ 0 (zero OR positive). For countdown loops, use BRP.

Self-Check Quiz

Q1: What does STA do?

STA stores the accumulator value into a specified mailbox (memory location).

Q2: What is the difference between BRA and BRZ?

BRA always branches (unconditional jump). BRZ only branches if the accumulator equals zero.

Q3: What does DAT do?

DAT reserves a mailbox for storing data. It can optionally initialise it with a value (e.g., DAT 1).
